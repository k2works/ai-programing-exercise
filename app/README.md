# テスト駆動開発から始めるJavaScript入門

ソフトウェア開発の三種の神器を準備する

## 概要

このプロジェクトは、テスト駆動開発（TDD）の手法を使ってJavaScriptプログラミングを学習するためのチュートリアルです。

## ソフトウェア開発の三種の神器

1. **バージョン管理システム（Git）** - コードの変更履歴を管理
2. **テスティングフレームワーク（Jest）** - 自動テストの実行
3. **タスクランナー（npm scripts）** - 開発タスクの自動化

## セットアップ

### 必要なソフトウェア

- Node.js (v18以上推奨)
- npm または yarn

### インストール

```bash
# 依存関係のインストール
npm install

# 開発用の依存関係も含めてインストール
npm install --save-dev
```

## 使用方法

### テストの実行

```bash
# テストを一回実行
npm test

# テストをウォッチモードで実行（ファイル変更時に自動実行）
npm run test:watch

# カバレッジレポート付きでテスト実行
npm run test:coverage
```

### アプリケーションの実行

```bash
# アプリケーションを起動
npm start

# 開発モード（ファイル変更時に自動再起動）
npm run dev
```

## プロジェクト構造

```
app/
├── src/                 # ソースコード
│   ├── index.js        # エントリーポイント
│   └── ...             # その他のソースファイル
├── test/               # テストファイル
│   └── ...             # テストコード
├── package.json        # プロジェクト設定
├── .babelrc           # Babel設定
└── README.md          # このファイル
```

## テスト駆動開発のサイクル

1. **Red** - 失敗するテストを書く
2. **Green** - テストを通すための最小限のコードを書く
3. **Refactor** - コードを改善する

## 学習ステップ

1. 環境構築とツールの準備
2. 最初のテストケース作成
3. TDDサイクルの実践
4. リファクタリング
5. 継続的インテグレーション
