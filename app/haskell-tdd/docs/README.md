# テスト駆動開発から始めるHaskell入門

## 概要

このプロジェクトは「テスト駆動開発から始めるRuby入門 ~ソフトウェア開発の三種の神器を準備する~」の内容をHaskellで実装したものです。

## ソフトウェア開発の三種の神器

### 1. バージョン管理
- Gitを使用してソースコードの変更履歴を管理
- コミットメッセージはAngularルールに従って記述

### 2. テスティング
- HSpecを使用した単体テスト
- テスト駆動開発（TDD）のアプローチを採用

### 3. 自動化
- Makefileによるタスクランナー
- `make watch`によるファイル変更の自動監視とテスト実行

## セットアップ手順

1. プロジェクトの初期セットアップ
```bash
make setup
```

2. プロジェクトのビルド
```bash
make build
```

3. テストの実行
```bash
make test
```

4. 自動テスト（開発時に使用）
```bash
make watch
```

## 実装内容

### FizzBuzzアプリケーション
- `src/FizzBuzz.hs`: FizzBuzzのロジックを実装
- `test/FizzBuzzSpec.hs`: FizzBuzzのテストケース
- `app/Main.hs`: メインアプリケーション

### 品質管理
- HLintによる静的コード解析
- `make lint`でコード品質をチェック
- `make format`で自動フォーマット

### Haskellの特徴を活かした実装
- 純粋関数型プログラミング
- パターンマッチングの活用
- 高階関数（map, filter, fold等）の使用
- 型安全性の確保

## コマンド一覧

| コマンド | 説明 |
|---------|------|
| `make help` | ヘルプを表示 |
| `make setup` | プロジェクトの初期セットアップ |
| `make build` | プロジェクトのビルド |
| `make test` | テストの実行 |
| `make lint` | 静的コード解析 |
| `make format` | コードフォーマット |
| `make watch` | ファイル変更を監視してテスト自動実行 |
| `make clean` | ビルド成果物の削除 |
| `make coverage` | カバレッジ付きテスト実行 |

## 開発フロー

1. `make watch`でファイル監視を開始
2. テストファイルを編集してテストケースを追加（Red）
3. 実装ファイルを編集してテストをパスさせる（Green）
4. `make lint`でコード品質をチェック
5. リファクタリングを実施（Refactor）
6. Gitでコミット

このサイクルを繰り返すことで、テスト駆動開発を実践できます。
