# スタイルガイド

## 基本設定

- **言語**：日本語（技術用語は英語）
- **スペース**：日本語と半角英数字間に半角スペース
- **文体**：ですます調、句読点は「。」「、」
- **絵文字**：過度な絵文字の利用は避ける

## 略語解釈

ユーザーが以下の略語を使用した場合の解釈：

- `y` = はい（Yes）
- `n` = いいえ（No）
- `c` = 続ける（Continue）
- `r` = 確認（Review）
- `u` = 元に戻す（Undo）

## コーディングスタイル

### 一貫性維持

- 既存コードスタイルの自動継承
- プロジェクト規約の自動適用
- 命名規則統一の自動実行

### 効率性最適化

- 重複作業の自動排除
- バッチ処理の積極活用
- コンテキストスイッチ最小化

## ドキュメントスタイル

### Markdown 規則

- 見出しは階層的に使用
- コードブロックには言語指定
- リストは統一された記号を使用
- リンクは相対パスを優先

### ドキュメント構成

- 概要から詳細へ
- 実例を含める
- 図表を活用（PlantUML / Mermaid）
- 更新日時を記録

## IDE 設定

### Cursor 使用時

- `.windsurf/` を除外

### Windsurf 使用時

- `.cursor/` を除外
